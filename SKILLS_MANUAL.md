# 富酬者AI引擎 - Skills命令手册

> **抖音AI职场课程配套教材** | 打造你的第二大脑

## 课程简介

富酬者AI引擎是一款基于 Obsidian 的智能知识管理工具，通过 iFlow CLI 提供强大的 AI 能力。本手册介绍所有可用的 Skills（技能命令），帮助你高效管理知识、创作内容、规划项目。

---

## 快速开始

### 基础命令格式

在 iFlow Chat 中输入 `/命令名称` 即可激活对应技能：

```
/开始一天      → 启动每日规划工作流
/爆款文案      → 生成视频口播脚本
/头脑风暴      → 进行创意探索
```

## 一、核心工作流 Skills

### 1. `/开始一天` - 每日规划工作流

**用途**：每天早晨启动，自动整理待办、获取资讯、规划今日重点

**工作流程**：
1. 回顾昨日未完成任务
2. 扫描活跃项目状态
3. 检查收件箱待处理项
4. 获取 AI 资讯和新闻摘要
5. 创建今日日记笔记

**输出示例**：
```
> 💡 每日一句：行动是治愈恐惧的良药

## 早安! 今日规划已就绪

**今日笔记:** [[2026-03-10]]

**待办事项:**
- [ ] 完成产品文档
- [ ] 回复客户邮件
- [ ] 准备周会材料

**正在进行项目 (2):**
- [[项目A]] - 开发中
- [[项目B]] - 规划阶段
```

**职场应用场景**：
- 每日晨间启动仪式
- 项目进度跟踪
- 时间管理优化

---

### 2. `/启动项目` - 项目立项

**用途**：将想法或收件箱笔记转化为结构化项目

**工作流程**：
1. 收集项目背景信息
2. 设计项目结构和里程碑
3. 创建项目笔记文件
4. 归档原始想法到收件箱

**输出格式**：
```markdown
---
title: 项目名称
type: project
status: active
priority: P1
---

# 项目名称

## 背景
[项目背景说明]

## 行动阶段
- 阶段1: [描述]
- 阶段2: [描述]

## 成功指标
- [ ] 指标1
- [ ] 指标2
```

**职场应用场景**：
- 新项目立项
- OKR 制定
- 任务拆解

---

### 3. `/深度研究` - 知识深挖

**用途**：对某个主题进行深度研究和知识整理

**工作流程**：
1. 创建研究计划
2. 搜索官方文档和资料
3. 创建主研究笔记
4. 提取原子概念到知识库

**输出结构**：
```
5-研究/领域/主题/
├── 主题.md          # 主笔记
├── examples/        # 示例代码
└── 相关概念链接

6-知识库/
├── 概念1.md
├── 概念2.md
```

**职场应用场景**：
- 技术学习
- 行业研究
- 竞品分析

## 二、内容创作 Skills

### 4. `/爆款文案` - 短视频脚本生成

**用途**：根据热门内容生成可直接使用的口播文案

**工作流程**：
1. 扫描收件箱偏好文件
2. 收集用户需求（内容方向、产品、受众）
3. 搜索小红书/抖音爆款内容
4. 分析爆款结构和话术
5. 生成口播逐字稿 + 拍摄建议

**输出格式**：
```
## 口播逐字稿（可直接复制进提词器）

你是不是每次化妆都要花半个小时打底，结果出门两小时就斑驳脱妆了？今天给你分享一个我用了三个月的底妆神器...

---

## 拍摄建议

场景：建议在光线充足的房间...
机位：手机竖屏拍摄...
表情：开头要有惊讶感...
```

**职场应用场景**：
- 短视频内容创作
- 直播带货脚本
- 社交媒体营销

---

### 5. `/头脑风暴` - 创意探索

**用途**：通过对话式探索深化想法，发现新方向

**工作流程**：
1. 理解初始想法和背景
2. 提出探索性问题
3. 挑战假设、拓展角度
4. 总结关键洞察
5. 提供下一步选择（创建项目/整理知识）

**对话风格**：
- 好奇且充满活力
- 支持但具有挑战性
- 关注可能性而非限制

**职场应用场景**：
- 产品创意发散
- 问题解决
- 战略规划

---

### 6. `/画布设计` - 视觉创作

**用途**：创建精美的海报、图表、信息图等视觉作品

**工作流程**：
1. 创建设计哲学（美学风格）
2. 确定主题概念
3. 生成 PNG 或 PDF 文件

**设计风格示例**：
- **Concrete Poetry**：大胆几何、纪念碑式排版
- **Chromatic Language**：色彩作为信息载体
- **Analog Meditation**：安静、质感、呼吸感

**输出**：
- `.png` 或 `.pdf` 视觉文件
- `.md` 设计哲学文档

**职场应用场景**：
- 社交媒体配图
- 演示文稿设计
- 品牌视觉设计

## 三、信息管理 Skills

### 7. `/快速问答` - 即时答疑

**用途**：快速回答问题，无需创建复杂笔记

**特点**：
- 直接回答，无冗余流程
- 可选择性保存到知识库
- 链接相关现有笔记

**示例**：
```
用户: /快速问答 什么是OKR？
AI: OKR (Objectives and Key Results) 是一种目标管理方法...
详见 [[OKR方法论]]
```

---

### 8. `/整理知识` - 知识结构化

**用途**：将文本内容整理为知识库结构

**工作流程**：
1. 分析文本主题和领域
2. 提取原子概念
3. 创建主笔记 + 知识库条目
4. 建立概念关联

**输出结构**：
```
5-研究/领域/主题/主题.md    # 主笔记
6-知识库/分类/概念1.md      # 原子概念
6-知识库/分类/概念2.md
```

**职场应用场景**：
- 读书笔记整理
- 培训资料归档
- 知识沉淀

---

### 9. `/归档` - 项目归档

**用途**：归档已完成的项目和已处理的收件箱

**工作流程**：
1. 查找已完成项目
2. 查找已处理收件箱条目
3. 移动到归档目录
4. 更新元数据

**归档结构**：
```
8-归档/
├── 项目/2026/
│   ├── ProjectA/
│   └── ProjectB.md
└── 收件箱/2026/03/
    └── idea-note.md
```

**用途**：创建符合 Obsidian 规范的 Markdown 文档

## 七、常见问题

### Q: Skills 和普通对话有什么区别？

**A**: Skills 是预设的工作流，具有固定的流程和输出格式，适合重复性任务。普通对话更灵活，适合开放性问题。

### Q: 如何自定义 Skill？

**A**: 在 `.agents/skills/` 目录下创建 `SKILL.md` 文件，按照以下格式：

```markdown
---
name: 技能名称
description: 技能描述
command: 命令
aliases: [别名1, 别名2]
---
[技能指令内容]
```

### Q: 文件保存在哪里？

**A**: 所有文件遵循 P.A.R.A 结构：
- `1-收件箱/` - 快速捕获
- `2-日记/` - 每日记录
- `3-项目/` - 活跃项目
- `4-计划/` - 规划文档
- `5-研究/` - 研究笔记
- `6-知识库/` - 永久知识
- `7-资源/` - 资源素材
- `8-归档/` - 已完成归档
