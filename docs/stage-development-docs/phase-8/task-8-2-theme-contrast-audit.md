# Task 8.2：审计深浅主题对比度与层级表达

## 所属阶段

阶段八：视觉精修与可访问性

## 目标

确保插件在深色和浅色主题下都具备足够对比度和清晰的视觉层级。

## 实施步骤

1. 列出重点 UI 元素。
2. 在两种主题下检查文字、背景和边框对比度。
3. 调整不达标区域。
4. 把可复用规则回收进样式变量或公共样式。

## 验收标准

1. 核心交互元素在两种主题下都清晰。
2. 不出现重要按钮与背景融在一起的问题。
3. 消息区层级表达稳定。

## 风险与注意事项

1. 不要为某一主题单独做过多硬编码补丁。
2. 需优先兼容 Obsidian 主题变量体系。
