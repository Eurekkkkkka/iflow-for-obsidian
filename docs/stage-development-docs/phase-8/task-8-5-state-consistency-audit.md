# Task 8.5：统一 hover、focus、active 状态规则

## 所属阶段

阶段八：视觉精修与可访问性

## 目标

让消息区、会话区、输入区中的交互控件在 hover、focus、active 三种状态下表现一致。

## 实施步骤

1. 盘点主要控件的交互状态样式。
2. 识别不一致或缺失状态。
3. 抽出统一规则并逐步替换。
4. 回归检查高频交互组件。

## 验收标准

1. 相同类型控件的交互状态风格统一。
2. 不会出现某些区域有反馈、某些区域无反馈的情况。
3. 用户对交互可点击性有更稳定预期。

## 风险与注意事项

1. 不要把所有控件做成同样视觉，仍需保留层级。
2. 状态样式调整要避免影响可读性。
