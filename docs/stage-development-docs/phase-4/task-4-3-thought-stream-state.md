# Task 4.3：在聊天视图建立独立思考流状态

## 所属阶段

阶段四：思考流与工具流拆分

## 目标

让聊天视图拥有独立的思考流状态和渲染容器，而不是继续把 thought chunk 拼接进最终回答正文。

## 实施步骤

1. 设计消息项中的思考流状态结构。
2. 明确思考流何时开始、何时结束、何时清理。
3. 将思考流与最终回答绑定到同一轮消息上下文。
4. 处理 onEnd 与异常结束场景。

## 验收标准

1. 思考流不会污染最终回答正文。
2. 每轮消息的思考内容归属清晰。
3. 异常中断时思考流状态也能正确收尾。

## 风险与注意事项

1. 必须避免引入新的流式同步问题。
2. 思考流状态不能破坏现有消息持久化逻辑。
