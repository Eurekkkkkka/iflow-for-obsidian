# Task 4.1：梳理 ACP 事件与显示语义

## 所属阶段

阶段四：思考流与工具流拆分

## 目标

明确 ACP 协议事件在 UI 中的显示语义，为后续拆分思考流和工具流打基础。

## 需要梳理的事件

1. `agent_message_chunk`
2. `agent_thought_chunk`
3. `tool_use`
4. `tool_result`
5. `task_finish`
6. `agent_turn_end`

## 实施步骤

1. 整理每类事件的来源和时序。
2. 定义每类事件在 UI 中属于哪一类信息。
3. 标记当前错误混流点。
4. 输出事件语义对照表。

## 验收标准

1. 每类事件都有清晰显示语义。
2. 事件语义可直接指导 mapper 改造。
3. 思考流、工具流、最终回答不再语义重叠。

## 风险与注意事项

1. 不要只按字段名猜测语义，需结合当前协议使用方式。
2. 语义定义必须能落到 UI 实现。
