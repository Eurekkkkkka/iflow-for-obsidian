# Task 4.6：处理思考流、工具流和正文的顺序与滚动行为

## 所属阶段

阶段四：思考流与工具流拆分

## 目标

确保流式更新时三类信息的顺序、归属和自动滚动行为稳定，不出现明显跳动和错位。

## 实施步骤

1. 定义三类信息在消息中的相对顺序。
2. 设计流式更新时的插入与更新策略。
3. 处理自动滚动和用户手动滚动的冲突。
4. 为长回复和多工具场景做走查。

## 验收标准

1. 消息更新顺序稳定。
2. 自动滚动不会干扰用户查看上文。
3. 长对话和多工具场景下消息区仍然可控。

## 风险与注意事项

1. 这是体验敏感区，优先稳定，再追求炫技动效。
2. 需要避免重排导致性能问题。
