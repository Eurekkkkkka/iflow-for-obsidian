# Task 4.4：定义思考流展示模式

## 所属阶段

阶段四：思考流与工具流拆分

## 目标

为思考流定义清晰的默认展示策略，兼顾可见性和消息区整洁度。

## 推荐模式

1. 隐藏。
2. 折叠摘要。
3. 完整展开。

## 实施步骤

1. 明确默认模式及切换入口。
2. 设计折叠摘要的文案和视觉样式。
3. 约定用户主动展开后的行为。
4. 为不同模式定义验收场景。

## 验收标准

1. 默认模式符合大多数用户阅读习惯。
2. 思考流展开与收起不会破坏消息区布局。
3. 相关交互在窄侧栏下依然可用。

## 风险与注意事项

1. 思考流不是主内容，视觉权重必须低于最终回答。
2. 不要让模式切换过于复杂。
