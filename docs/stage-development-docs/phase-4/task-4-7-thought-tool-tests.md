# Task 4.7：补思考流与工具流测试和验收场景

## 所属阶段

阶段四：思考流与工具流拆分

## 目标

为阶段四的事件拆分和 UI 展示建立测试护栏，并形成可复用的人工验收场景。

## 建议覆盖点

1. `agent_thought_chunk` 单独映射。
2. 工具流状态变化。
3. 最终回答与思考流不混流。
4. onEnd 与异常结束清理。
5. 折叠和展开行为。

## 验收标准

1. 关键事件拆分逻辑有自动化测试。
2. UI 行为有清晰人工验收步骤。
3. 阶段四改造后能稳定回归。

## 风险与注意事项

1. 测试应覆盖协议层与视图层交界。
2. 不要只验证 happy path，必须覆盖异常结束场景。
