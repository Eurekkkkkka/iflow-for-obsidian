# Task 4.5：定义工具流展示策略

## 所属阶段

阶段四：思考流与工具流拆分

## 目标

让工具调用展示更可读，既保留执行透明度，也避免占满消息区。

## 推荐策略

1. 运行中展开。
2. 完成后折叠为摘要。
3. 失败时保留关键错误信息和重试方向。

## 实施步骤

1. 梳理当前工具调用展示状态。
2. 设计运行中与完成后的差异化展示。
3. 处理结果过长时的折叠逻辑。
4. 为错误态单独定义文案和视觉样式。

## 验收标准

1. 工具调用过程仍然可见。
2. 工具调用完成后不会淹没正文内容。
3. 失败时能快速看到原因和影响范围。

## 风险与注意事项

1. 不要丢失关键执行信息。
2. 工具结果折叠必须支持再次展开查看。
