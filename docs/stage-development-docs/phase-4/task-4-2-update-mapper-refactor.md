# Task 4.2：改造事件映射层

## 所属阶段

阶段四：思考流与工具流拆分

## 目标

将当前把思考流和最终回答都映射为普通 stream 的逻辑拆开，建立明确的事件类型边界。

## 涉及文件

1. `src/features/chat/toolCallMapper.ts`
2. `src/iflowService.ts`
3. 相关测试文件

## 实施步骤

1. 为消息流、思考流、工具流定义独立事件类型。
2. 改造 mapper 输出结构。
3. 保持现有调用链尽量兼容。
4. 为新旧路径补测试。

## 验收标准

1. `agent_thought_chunk` 不再映射成普通正文流。
2. 工具事件不会被正文渲染链吞掉。
3. 上层 UI 可以基于事件类型独立展示。

## 风险与注意事项

1. 这是协议和 UI 的交界层，必须谨慎控制兼容性。
2. 不要在这个任务里顺手重写过多上层逻辑。
