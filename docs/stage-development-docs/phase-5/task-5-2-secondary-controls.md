# Task 5.2：将模型、模式和思考开关下沉为二级入口

## 所属阶段

阶段五：顶部信息架构重组

## 目标

把低频但重要的控制项从顶部常驻区降级为二级入口，释放主区空间。

## 实施步骤

1. 定义二级入口形式，例如弹层、popover 或设置面板。
2. 设计二级入口的触达方式。
3. 保留必要的状态摘要，避免用户完全失去上下文。
4. 验证切换模型、模式、思考开关的效率是否可接受。

## 验收标准

1. 顶部主区明显变简洁。
2. 用户仍能快速找到模型和模式配置。
3. 当前选中状态仍然可感知。

## 风险与注意事项

1. 下沉不等于隐藏，仍需保证可发现性。
2. 不要把所有控制项都塞进设置页。
