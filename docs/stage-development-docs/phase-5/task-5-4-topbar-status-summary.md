# Task 5.4：为顶部栏补关键状态摘要

## 所属阶段

阶段五：顶部信息架构重组

## 目标

在减少顶部控件的同时，保留必要的状态摘要，让用户不丢失当前上下文。

## 可选状态摘要

1. 当前模型摘要。
2. 当前连接状态。
3. 当前会话状态。
4. 思考模式是否开启。

## 实施步骤

1. 选择最值得保留的摘要项。
2. 设计紧凑的展示方式。
3. 明确摘要项是否可点击进入配置。
4. 控制摘要信息密度。

## 验收标准

1. 用户能快速感知当前关键状态。
2. 摘要信息不会重新把顶部栏塞满。
3. 摘要与二级入口之间关系清晰。

## 风险与注意事项

1. 摘要应强调可读性，避免又变成迷你控制面板。
2. 状态过多会让重构失去意义。
