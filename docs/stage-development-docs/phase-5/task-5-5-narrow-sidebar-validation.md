# Task 5.5：验证窄侧栏下顶部栏布局稳定性

## 所属阶段

阶段五：顶部信息架构重组

## 目标

确保顶部重构后的布局在 Obsidian 窄侧栏下依然稳定，不溢出、不遮挡、不挤压主内容。

## 实施步骤

1. 选择多个典型宽度进行验证。
2. 检查标题截断、按钮换行、状态挤压等问题。
3. 根据结果调整布局优先级和最小宽度策略。
4. 记录需要在阶段八处理的视觉细节。

## 验收标准

1. 顶部栏在窄侧栏下仍可操作。
2. 主要入口不会因为缩窄而消失。
3. 消息区和输入区不受顶部栏重构负面影响。

## 风险与注意事项

1. 侧栏宽度变化是 Obsidian 的常见场景，不能忽略。
2. 先保证可用，再考虑美观。
