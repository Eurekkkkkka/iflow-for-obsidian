# Task 3.2：补 Markdown 表格支持

## 所属阶段

阶段三：消息内容渲染与阅读质量

## 目标

支持模型输出的 Markdown 表格，避免当前完全无法渲染或结构混乱。

## 实施步骤

1. 评估现有渲染器是否足以支持表格。
2. 定义最小可用表格语法范围。
3. 接入表格 HTML 输出与样式。
4. 为对齐、空单元格、窄宽度场景补测试。

## 验收标准

1. 常规 Markdown 表格可读。
2. 窄侧栏下表格不会完全破坏布局。
3. 样式与消息正文视觉语言保持一致。

## 风险与注意事项

1. 不必追求完整 Markdown 规范，先保证高频场景可用。
2. 表格样式不能导致横向滚动失控。
