# Task 3.5：审计渲染元素在深浅主题下的可读性

## 所属阶段

阶段三：消息内容渲染与阅读质量

## 目标

确保代码块、行内代码、文件路径高亮、表格和 Thinking 块在深浅主题下都有足够可读性。

## 实施步骤

1. 列出所有消息渲染元素。
2. 在深色主题和浅色主题下逐项检查。
3. 调整颜色、边框和背景对比度。
4. 记录需要在阶段八统一收敛的样式债务。

## 验收标准

1. 主要渲染元素在两种主题下都清晰可辨。
2. 不出现文字和背景对比不足的问题。
3. 消息区视觉层级保持稳定。

## 风险与注意事项

1. 优先使用 Obsidian 变量，不要硬编码主题颜色。
2. 此任务关注可读性，不追求最终视觉精修。
