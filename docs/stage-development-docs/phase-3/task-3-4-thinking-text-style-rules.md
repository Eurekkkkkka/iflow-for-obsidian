# Task 3.4：修正 Thinking 文本样式匹配规则

## 所属阶段

阶段三：消息内容渲染与阅读质量

## 目标

降低 Thinking 样式对固定文案格式的依赖，避免模型稍微换一种写法就失效。

## 实施步骤

1. 盘点当前 Thinking 文本匹配规则。
2. 区分协议级思考流与普通文本中的 Thinking 文案。
3. 定义更稳健的文本样式触发条件。
4. 为正例和反例补测试。

## 验收标准

1. 普通文本中的提示性 Thinking 区块可以稳定渲染。
2. 不会误把普通段落错误标成 Thinking。
3. 与阶段四的思考流拆分方案不冲突。

## 风险与注意事项

1. 文本样式规则不能替代协议级思考流设计。
2. 需要避免正则过宽导致误匹配。
