# Task 7.3：增加 SDK 重启与重新检测入口

## 所属阶段

阶段七：设置页与诊断闭环

## 目标

让用户在连接异常时可以直接执行重启或重新检测，而不需要离开当前界面自行猜测处理方式。

## 实施步骤

1. 明确重启、重新检测两类操作的语义。
2. 设计入口位置和触发条件。
3. 为操作执行过程补状态提示。
4. 处理成功和失败后的反馈。

## 验收标准

1. 用户能直接发起恢复动作。
2. 操作结果有明确反馈。
3. 不需要依赖命令面板才能完成基本恢复。

## 风险与注意事项

1. 要清楚区分“重启 SDK”和“重新检测连接”。
2. 需避免重复触发造成状态混乱。
