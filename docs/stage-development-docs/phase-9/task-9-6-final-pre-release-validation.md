# Task 9.6：完成发布前三类最终验收

## 所属阶段

阶段九：测试、文档与发布准备

## 目标

在发布前完成三类关键环境的最终验收，降低“本地看起来没问题，用户环境出问题”的风险。

## 必验场景

1. 深色主题。
2. 浅色主题。
3. 窄侧栏。

## 实施步骤

1. 在三类环境中走查核心用户旅程。
2. 重点检查顶部栏、消息区、上下文区、输入区和思考流工具流展示。
3. 记录阻断级和非阻断级问题。
4. 输出最终发布前结论。

## 验收标准

1. 三类环境中的核心用户旅程全部通过。
2. 无阻断级问题遗留。
3. 非阻断问题已记录并可后续跟进。

## 风险与注意事项

1. 最终验收必须站在用户视角，而不是开发自测视角。
2. 如果发现阻断问题，应停止发布并回流到对应阶段处理。
