# Task 9.2：为思考流拆分补事件映射与 UI 测试

## 所属阶段

阶段九：测试、文档与发布准备

## 目标

确保阶段四引入的思考流拆分在协议层和 UI 层都有测试护栏。

## 实施步骤

1. 为事件映射逻辑补单元测试。
2. 为思考流与正文分离行为补视图级测试。
3. 覆盖异常结束和边界场景。
4. 把结果纳入发布前必跑集。

## 验收标准

1. `agent_thought_chunk` 的处理行为有明确测试。
2. 最终回答与思考流不混流有测试保障。
3. 关键异常路径不再只靠人工记忆验证。

## 风险与注意事项

1. 需同时覆盖 mapper 和 chatView 两层。
2. 不要只测 happy path。
