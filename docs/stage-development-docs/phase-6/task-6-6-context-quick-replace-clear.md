# Task 6.6：为上下文区增加快速替换或清空能力

## 所属阶段

阶段六：高频交互效率优化

## 目标

让上下文管理从“只能看到”升级到“可以快速调整”，减少发送前修改成本。

## 实施步骤

1. 明确支持清空哪些上下文类型。
2. 定义替换或重新附加的最小交互路径。
3. 校验状态更新是否与发送链路一致。
4. 评估是否需要确认提示。

## 验收标准

1. 用户能快速清空不需要的上下文。
2. 清空或替换后内部状态同步正确。
3. 不会打断正常发送主流程。

## 风险与注意事项

1. 上下文调整是发送前高频动作，操作必须轻量。
2. 需要防止自动附加逻辑立即把用户刚移除的内容重新加回。
