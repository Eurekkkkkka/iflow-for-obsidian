# Task 1.1：整理核心用户旅程

## 所属阶段

阶段一：体验基线与验收标准

## 目标

把插件当前最重要的 6 条用户旅程整理成统一文档，作为后续所有优化任务的体验基线。

## 覆盖范围

1. 首次启动与 SDK 连接。
2. 普通文本提问。
3. 附带当前文件提问。
4. 附带选中文本提问。
5. 附带图片提问。
6. 工具调用失败后的恢复。

## 涉及文件

1. `README.md`
2. `src/main.ts`
3. `src/chatView.ts`
4. `src/features/chat/chatContextBuilder.ts`
5. `src/iflowService.ts`

## 实施步骤

1. 梳理每条旅程的进入条件。
2. 标出用户可见的主要反馈节点。
3. 标出关键分叉和退出条件。
4. 将结果沉淀为统一旅程图或步骤表。

## 交付物

1. 一份旅程清单。
2. 每条旅程的关键触点说明。
3. 每条旅程的主要痛点摘要。

## 验收标准

1. 6 条旅程均有完整入口、过程、结果描述。
2. 每条旅程都能映射到当前代码入口。
3. 后续任务可直接引用该文档，不再重复梳理路径。

## 风险与注意事项

1. 不要把技术实现细节混入用户旅程主文。
2. 旅程必须基于当前真实功能，而不是期望功能。
