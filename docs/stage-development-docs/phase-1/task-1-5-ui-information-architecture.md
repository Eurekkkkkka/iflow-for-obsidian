# Task 1.5：整理当前 UI 信息架构图

## 所属阶段

阶段一：体验基线与验收标准

## 目标

明确当前聊天视图各区域的职责边界，为后续顶部重构、上下文重构和消息区拆分提供依据。

## 需要覆盖的区域

1. 顶部栏。
2. 控制栏。
3. 上下文区。
4. 消息区。
5. 输入区。
6. 会话面板。

## 实施步骤

1. 画出当前视图结构图。
2. 标注每个区域的职责。
3. 标注每个区域承载的信息类型。
4. 标注已知拥挤、冲突、难发现的问题。

## 交付物

1. 一份当前 UI 信息架构图。
2. 一份职责边界说明。
3. 一份主要问题清单。

## 验收标准

1. 各区域职责边界明确。
2. 能支持阶段五的顶部架构重组。
3. 能支持阶段二、四、六的局部优化拆分。

## 风险与注意事项

1. 信息架构图要基于当前实现，而不是理想结构。
2. 问题描述应尽量落到“用户为什么看不懂”或“为什么难操作”。
