# Task 1.3：整理失败分支与恢复路径

## 所属阶段

阶段一：体验基线与验收标准

## 目标

列出插件主要失败场景，并为每个场景定义最小恢复路径。

## 覆盖失败场景

1. SDK 启动失败。
2. 连接超时或连接中断。
3. 图片过大或格式不支持。
4. 工具调用失败。
5. 无历史会话。
6. localStorage 接近上限或写入失败。

## 实施步骤

1. 收集当前已知失败场景。
2. 记录现有反馈方式。
3. 记录用户当前可执行的恢复动作。
4. 标记恢复能力缺口。

## 交付物

1. 失败场景清单。
2. 当前反馈与缺口对照表。
3. 恢复动作建议表。

## 验收标准

1. 主要失败场景覆盖完整。
2. 每个失败场景至少定义一个可执行恢复动作。
3. 阶段二和阶段七可以直接复用该结果。

## 风险与注意事项

1. 恢复动作必须简单明确。
2. 不要把诊断建议写成开发者术语。
