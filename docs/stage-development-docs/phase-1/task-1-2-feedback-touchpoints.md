# Task 1.2：补齐旅程关键反馈节点

## 所属阶段

阶段一：体验基线与验收标准

## 目标

为每条用户旅程标注关键反馈节点，明确用户在每一步应看到什么状态。

## 核心问题

当前插件拥有较多运行时能力，但反馈分散在 Notice、流式文本和局部控件中，用户很难形成稳定预期。

## 实施步骤

1. 逐条旅程列出关键节点。
2. 为每个节点定义预期反馈类型。
3. 标记当前已有反馈和缺失反馈。
4. 输出一张反馈矩阵。

## 反馈类型建议

1. 状态提示。
2. 上下文提示。
3. 成功反馈。
4. 失败反馈。
5. 下一步操作提示。

## 验收标准

1. 每条旅程的关键节点都有预期反馈定义。
2. 能明确看出哪些节点需要在阶段二优先补齐。
3. 文档可直接作为 UI 文案和状态设计的输入。

## 风险与注意事项

1. 反馈定义必须站在用户视角，而不是日志视角。
2. 避免把多个状态合并成模糊描述。
