# Task 1.4：建立阶段验收清单

## 所属阶段

阶段一：体验基线与验收标准

## 目标

为后续每个阶段建立统一的人工验收清单，避免只做代码修改而忽略真实体验回归。

## 实施步骤

1. 从用户旅程中提取关键验收点。
2. 按阶段建立验收项。
3. 区分功能验收、UI 验收、错误恢复验收。
4. 为每项验收定义通过标准。

## 清单分类建议

1. 启动与连接。
2. 输入与发送。
3. 上下文与附件。
4. 消息渲染。
5. 思考流与工具流。
6. 会话管理。
7. 错误恢复。

## 验收标准

1. 每个后续阶段至少能映射到一组验收项。
2. 验收语言可被开发和产品共同理解。
3. 清单可以作为发布前回归基线。

## 风险与注意事项

1. 验收项不要过度抽象。
2. 每一项都必须能被明确判定通过或不通过。
