# Task 2.4：重做上下文摘要区

## 所属阶段

阶段二：运行时反馈与上下文可见性

## 目标

把当前零散的上下文提示重组为清晰的发送前摘要，让用户知道模型将接收到哪些内容。

## 要显示的上下文类型

1. 自动附加文件。
2. 手动附加文件。
3. 选中文本。
4. 图片附件。

## 实施步骤

1. 定义上下文摘要区的数据结构。
2. 设计摘要区的显示规则。
3. 为不同来源提供视觉区分。
4. 支持无上下文时的收起或空态处理。

## 验收标准

1. 用户在发送前能明确知道本轮上下文内容。
2. 自动附加与手动附加有清晰区分。
3. 摘要区不侵占过多垂直空间。

## 风险与注意事项

1. 只展示摘要，不要一次性展开完整内容。 
2. 视觉权重不能压过消息主区域。
