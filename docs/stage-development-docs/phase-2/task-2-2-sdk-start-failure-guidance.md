# Task 2.2：增强 SDK 启动失败诊断提示

## 所属阶段

阶段二：运行时反馈与上下文可见性

## 目标

当 SDK 启动失败时，给用户明确原因范围和下一步操作，而不是只提示“启动失败”。

## 实施步骤

1. 区分常见失败类型，例如未安装 CLI、端口不可用、认证失败、启动超时。
2. 为每种失败定义用户可执行动作。
3. 约束错误提示的长度和展示层级。
4. 确保 Notice 和运行态状态区文案一致。

## 涉及文件

1. `src/main.ts`
2. `src/iflowService.ts`
3. `src/i18n/zh-CN.ts`
4. `src/i18n/en-US.ts`

## 验收标准

1. 启动失败时至少提供一条可执行建议。
2. 用户无需看开发者日志也能知道下一步怎么做。
3. 提示文案不暴露不必要的底层细节。

## 风险与注意事项

1. 不要承诺无法自动检测的具体原因。
2. 先覆盖最常见失败场景，再逐步细化。
