# Task 2.1：建立运行时状态文案体系

## 所属阶段

阶段二：运行时反馈与上下文可见性

## 目标

统一插件在启动、连接、发送和失败过程中的状态文案，减少用户理解成本。

## 状态集合

1. 检测中。
2. 已连接。
3. 启动中。
4. 启动失败。
5. 认证失败。
6. 发送中。
7. 已完成。
8. 已超时。

## 实施步骤

1. 盘点当前分散在 Notice、按钮和消息区中的状态文案。
2. 统一命名和语气。
3. 定义每个状态的展示位置。
4. 为文案预留 i18n 接入点。

## 验收标准

1. 同一种状态不再出现多个不同表达。
2. 状态名称可以被产品和开发共同复用。
3. 后续 UI 实现可以直接按该文案体系接入。 

## 风险与注意事项

1. 文案必须面向用户，而不是面向日志。 
2. 状态命名要避免歧义。
