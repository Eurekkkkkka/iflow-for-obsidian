# Task 2.9：建立用户可见错误提示规范

## 所属阶段

阶段二：运行时反馈与上下文可见性

## 目标

为用户可见错误建立统一规范，避免把原始异常、堆栈或技术术语直接暴露给普通用户。

## 规范内容

1. 错误分级。
2. 错误标题。
3. 错误说明。
4. 下一步操作。
5. 是否需要保留技术细节入口。

## 实施步骤

1. 收集现有错误显示方式。
2. 定义用户错误消息模板。
3. 明确哪些错误可直接展示，哪些需要降噪处理。
4. 与 i18n 文案和 Notice 机制对齐。

## 验收标准

1. 用户可见错误均符合统一结构。
2. 不再直接显示原始异常对象。
3. 错误提示至少包含建议动作或恢复方向。

## 风险与注意事项

1. 不要过度抽象导致开发排查困难。
2. 如需保留技术细节，应放在次级信息层。
