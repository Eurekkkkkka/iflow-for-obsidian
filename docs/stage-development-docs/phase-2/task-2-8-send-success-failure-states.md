# Task 2.8：统一发送中、成功、失败、超时状态

## 所属阶段

阶段二：运行时反馈与上下文可见性

## 目标

把消息发送相关状态统一起来，避免用户只在某些场景看到加载动画，在另一些场景只看到裸错误。

## 实施步骤

1. 定义发送状态状态机。
2. 统一消息级和视图级状态展示。
3. 为超时与失败区分不同文案和视觉语义。
4. 明确成功结束时的清理行为。

## 验收标准

1. 同一次发送过程中状态切换清晰可见。
2. 超时与失败不再表现为相同结果。
3. 状态切换不会导致消息区明显抖动。

## 风险与注意事项

1. 不要引入复杂状态机而影响现有主流程稳定性。
2. 发送状态必须与 streaming 生命周期严格一致。
