# Task 2.7：补齐空状态与搜索无结果引导

## 所属阶段

阶段二：运行时反馈与上下文可见性

## 目标

改善会话列表的空状态和搜索无结果状态，让用户知道下一步怎么操作。

## 实施步骤

1. 区分“还没有对话”和“搜索无结果”两种状态。
2. 为无历史会话状态提供新建引导。
3. 为搜索无结果状态提供清空搜索建议。
4. 统一空状态文案风格。

## 验收标准

1. 会话列表空状态不再只有一句静态文案。
2. 用户可以直接从空状态进入下一步动作。
3. 文案与顶部会话入口保持一致。

## 风险与注意事项

1. 空状态应简洁，不要堆砌说明文字。
2. 新建入口应明显但不喧宾夺主。
