# Task 2.3：在聊天视图中增加连接状态展示

## 所属阶段

阶段二：运行时反馈与上下文可见性

## 目标

把连接状态从一次性 Notice 扩展为聊天视图内可持续感知的轻量状态。

## 实施步骤

1. 设计连接状态的展示位置。
2. 明确何时展示、何时弱化、何时隐藏。
3. 接入连接中、已连接、失联、重试中等状态。
4. 与顶部栏或上下文区保持一致的视觉语言。

## 验收标准

1. 用户打开聊天视图时能快速判断当前连接状况。
2. 状态变化不会抢占主要聊天内容焦点。
3. 与现有 Notice 不形成冲突或重复噪音。

## 风险与注意事项

1. 状态展示必须足够轻量。
2. 避免持续闪烁或频繁变化造成干扰。
